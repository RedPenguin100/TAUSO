# ASO patent normalizer annotations

What normalizing/reference gene (or basis) each source patent used to quantify **in-vitro target-mRNA knockdown** in the TAUSO/OligoAI dataset (`../aso_inhibitions_with_canonical_gene.csv.gz`, column `patent_id`, 343 unique patents).

**Why:** ASO knockdown is reported as % of control; the normalizer determines whether a measured drop reflects true target-specific knockdown. The common academic assumption is ACTB (β-actin). This curation tests that against the primary literature.

## Coverage (as of 2026-05-26) — 100%
- **343 / 343 patents → 173,191 / 173,191 rows = 100%.**
- **114 hand-curated** (agents reading the patent Examples; the high-volume head + sampled tail) → authoritative.
- **229 auto-classified** by `scripts/classify_normalizer.py` reading the cached HTML (the homogeneous early-2000s Isis tail, ≤80 rows each). The classifier agrees with the manual labels on **86.7%** of the 114 — and is deliberately *conservative*: its disagreements are almost all "manual total-RNA → auto unknown", never inventing an exotic normalizer.
- Two masters: `patent_normalizer_annotations.csv` (114 manual, highest quality) and `patent_normalizer_annotations_full.csv` (all 343, with a `source` column = manual_curation | auto_classified).

## Headline result — ACTB is essentially never the normalizer

Full coverage (343 patents), counted by **patents** / by **rows of data**:

| normalization_basis | patents | rows | notes |
|---|---|---|---|
| **total RNA via RiboGreen** | 109 | ~154k (88.9%) | the Ionis/Isis house standard (RT-qPCR, "% UTC") |
| housekeeping gene | 9 | 3,928 (2.3%) | 1× **GUSB** (ATXN2), 8× **GAPDH** (early-2000s Isis + CD40) |
| unknown | 225 | 15,265 (8.8%) | early-2000s Isis "GAPDH *or* total RNA via RiboGreen" boilerplate — uncommitted, **but still NOT ACTB** |

- **ACTB used as the normalizer: 1 of 343 patents** — `US20180273577A1` (KRAS), 2,504 rows (1.4% of data), and only for a *minority* of in-vitro A431 dose-response blocks (predominant basis there is still RiboGreen). Every other patent is `used_actb = no` for its dataset cell line.
- A few more patents (STAT3, IRF5 ×2, MALAT1 ×2) *mention* β-actin as a normalizer, but only for off-dataset contexts (e.g. KARPAS-229 xenograft, MDA-MB-436) — verified by hand; not the dataset rows.
- All patents use **RT-qPCR** (no bDNA/QuantiGene found, even in 2002–2005 patents).
- The 225 "unknown" matter for the ACTB question only in that they are **not** ACTB: the patent text limits them to {GAPDH, total RNA}.

### Important caveat: β-actin/GAPDH/cyclophilin decoys
A keyword hit for these is usually NOT the normalizer for our rows. They appear as (a) in-vivo **mouse** study normalizers (mouse cyclophilin A / mouse GAPDH), (b) generic "either a housekeeping gene … or total RNA via RiboGreen" boilerplate, (c) western-blot **protein** loading controls, or (d) Northern-blot confirmatory assays — none of which is the in-vitro RT-qPCR readout the dataset rows come from. Each annotation traces the hit to the specific assigned cell line. Genuine in-vitro mixed cases: SCN2A (`US20240026353A1`, ≈50/50 RiboGreen vs GAPDH in SH-SY5Y).

## How the fetching was done (every fetch is logged)
Method is recorded per patent in the `fetch_method` column and in `fetch_log.csv`.

| fetch_method | patents | how |
|---|---|---|
| `google_patents_html_grep` | 109 | `curl -sL https://patents.google.com/patent/<id>/en` → grep the OCR'd full-text Examples section. (Plain `WebFetch` of the page is useless — it truncates before Examples.) |
| `uspto_image_pdf_ocr` | 4 | image-only scanned patents: download USPTO image PDF, render Examples pages with `pdftoppm`, read as images |
| `granted_sibling_html` | 0 | (used in-line in a couple notes where a pre-grant pub was image-only) |

Full methodology is captured as a reusable skill: `.claude/skills/aso-patent-normalizer/SKILL.md`.

## Files in this folder
- `patent_normalizer_annotations_full.csv` — **100%-coverage master** (343 patents, `source` = manual_curation | auto_classified). From `scripts/classify_normalizer.py`.
- `classified_normalizers.csv` — auto-classifier output for all 343 (basis + evidence sentence + ACTB flag); the audit behind the auto labels.
- `SUMMARY_full.txt` — full-coverage tallies incl. the ACTB count.
- `patent_normalizer_annotations.csv` — **manual master** (114 patents), highest quality, 13 columns (incl. verbatim `supporting_quote` + `source_url`). Regenerated by `scripts/consolidate.py`.
- `fetch_log.csv` — provenance log: `patent_id, target_gene, n_rows, fetch_method, source_url, confidence`.
- `evidence.csv` — **where each finding lives in its source**: `cache_file, quote_found, char_offset, example_heading, keyword_hits, context_snippet`. From `scripts/extract_evidence.py`.
- `evidence/<PATENT_ID>.md` — per-patent: the recorded quote in surrounding context near its `Example N`, plus every normalization-keyword hit with offset + example (raw audit trail).
- `html_cache/<PATENT_ID>.html.gz` — cached full-text patent HTML (the source). Makes re-runs instant; `scripts/fetch_patents.py` skips anything already cached.
- `SUMMARY.txt` — auto-generated tallies (incl. the ACTB count).
- `batch_1.csv … batch_22.csv` — raw per-agent batch outputs (audit trail).
- `scripts/` — the reproducible/resumable pipeline; see `scripts/README.md`.

## Reproduce / resume (fast)
```bash
PY=/home/michael/miniforge3/envs/tauso/bin/python
$PY scripts/consolidate.py        # batches -> manual master + fetch_log + SUMMARY
$PY scripts/fetch_patents.py --top 343   # cache source HTML for all 343 (skips already-cached)
$PY scripts/extract_evidence.py   # locate every quote in its source -> evidence.csv + evidence/
$PY scripts/classify_normalizer.py # auto-classify the tail -> full master + SUMMARY_full + validation
```
Because the HTML cache and master CSV persist on disk, a re-ask of this task re-uses them and only does new work — see `scripts/README.md`.

## Columns (master)
`patent_id, target_gene, cell_line, n_rows, normalizer_gene, normalization_basis, assay_method, used_actb, confidence, fetch_method, source_url, supporting_quote, notes`
- `normalization_basis` ∈ {total_RNA_ribogreen, housekeeping_gene, spike_in, unknown}
- `assay_method` ∈ {RT-qPCR, bDNA_QuantiGene, RNA-seq, other, unknown}
- `used_actb` ∈ {yes, no, unclear}
- `confidence` ∈ {high, medium, low, unknown} — high/medium always backed by a verbatim `supporting_quote`
